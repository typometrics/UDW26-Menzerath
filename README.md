# Typometrics: Cross-Linguistic Dependency Analysis Pipeline

A comprehensive analysis pipeline for extracting and visualizing typometric patterns from [Universal Dependencies](https://universaldependencies.org/) treebanks, focusing on verb-centered dependency structures, constituent ordering, and cross-linguistic variation.

This code accompanies the paper:

> Pegah Faghiri, Kim Gerdes, Sylvain Kahane (2026). *Verifying the Menzerath-Altmann law in the verbal domain in 180 languages.* UDW26 @ LREC 2026.

## Reproducing the Paper Results

This section describes the **minimum steps** needed to reproduce the figures, tables, and statistics reported in the paper. The pipeline has many additional exploratory notebooks and modules; only the ones marked ⭐ below are required for the paper.

### What the paper uses

The paper contains:

| Paper element | Source | Generated by |
|---|---|---|
| **Appendix A** — Table of all languages with MAL/LMAL/RMAL mini-plots and β values | `latex/mal_miniplots_table.tex` | `scripts/generate_latex_mal_miniplots.py` |
| **Figures 1–3** — Log-log MAL plots (German, Arabic, Czech) | Inline TikZ in `section4-metrics.tex` | Notebook 08 (data); manually formatted |
| **Figures 4–6** — Anti-MAL examples (Old East Slavic, Naija) | Inline TikZ in `section5-52-zoom.tex` | Notebook 08 (data); manually formatted |
| **Figure β(1→2) vs β(2→∞)** — Scatter plot by language family | Inline TikZ in `section4-metrics.tex` | Notebook 08 |
| **Tables 1–4** — MAL/LMAL/RMAL type counts by family and VO/OV type | `section5-results.tex` | Notebook 08 (statistics) |
| **All inline statistics** — β values, R² values, compliance ratios, Fisher's exact tests, language counts | `section4-metrics.tex`, `section5-results.tex` | Notebook 08 |
| **Interactive HTML reports** — Sortable tables with inline log-log plots | `html_analyses/mal_n_analysis*.html` | Notebook 08 → `src/mal_html_report.py` |

### Minimum pipeline (3 steps)

```
Step 1: Data preparation & extraction
   Notebook 01 → Notebook 04    (or Notebook 02)
       │                │
       ▼                ▼
   metadata.pkl    position2sizes.pkl, position2num.pkl, vo_vs_hi_scores.csv

Step 2: MAL analysis & HTML reports
   Notebook 08
       │
       ▼
   html_analyses/mal_n_analysis*.html  +  all statistics for paper

Step 3: LaTeX appendix table
   python scripts/generate_latex_mal_miniplots.py
       │
       ▼
   latex/mal_miniplots_table.tex
```

#### Step 1 — Data extraction (⭐ Notebooks 01 + 04)

Download [UD treebanks v2.17](https://universaldependencies.org/) and place them in `ud-treebanks-v2.17/`. Then run:

- **Notebook 01** (`01_data_preparation_and_validation.ipynb`) — Validates language codes, creates 10k-sentence chunks in `2.17_short/`, saves `data/metadata.pkl`.
- **Notebook 04** (`04_data_processing.ipynb`) — Runs `conll_processing.get_all_stats_parallel()` to extract dependency sizes, VO scores, bastard statistics, and disorder from all treebanks. Saves the key intermediate files:

| Intermediate file | Contents | Used by |
|---|---|---|
| `data/metadata.pkl` | Language names, families, colors | Notebooks 04, 08; LaTeX script |
| `data/all_langs_position2sizes.pkl` | Sum of constituent sizes per position/n/language | Notebook 08; LaTeX script |
| `data/all_langs_position2num.pkl` | Occurrence counts per position/n/language | Notebook 08; LaTeX script |
| `data/vo_vs_hi_scores.csv` | VO score (0–1) and head-initiality per language | Notebook 08; LaTeX script |

These four files are all that is needed for the paper outputs.

#### Step 2 — MAL analysis (⭐ Notebook 08)

Run **Notebook 08** (`08_menzerath_altmann_analysis.ipynb`). This notebook:

- Computes MAL_n, LMAL_n, RMAL_n for each language (constituent size for n dependents, minimum 100 tokens per configuration)
- Fits log-log regressions to get β slopes and R² values
- Classifies languages: MAL (β > 0.1), Anti-MAL (β < −0.1), grey zone (|β| ≤ 0.1)
- Computes the MAL compliance ratio (proportion of consecutive decreasing MAL_n pairs)
- Splits results by language family (IE vs non-IE) and word order type (VO > 0.67, OV < 0.33, NDO)
- Performs Fisher's exact tests for category independence
- Generates interactive HTML reports via `src/mal_html_report.py`

**Outputs:**
- `html_analyses/mal_n_analysis.html` — All languages
- `html_analyses/mal_n_analysis_vo_languages.html` — VO languages only
- `html_analyses/mal_n_analysis_ov_languages.html` — OV languages only
- `html_analyses/mal_n_analysis_mixed_languages.html` — Mixed/NDO languages
- All inline statistics cited in the paper (β values, counts, percentages, p-values)

#### Step 3 — LaTeX appendix table (⭐ `generate_latex_mal_miniplots.py`)

```bash
python scripts/generate_latex_mal_miniplots.py
```

This script loads the same four data files and produces `latex/mal_miniplots_table.tex` — a `longtable` with inline TikZ mini-plots for every language, color-coded green (MAL), red (Anti-MAL), or orange (grey zone).

### Key thresholds and parameters

| Parameter | Value | Description |
|---|---|---|
| Minimum token count | 100 | Configurations with < 100 occurrences are excluded from β regression |
| MAL threshold | β > 0.1 | Language classified as MAL-compliant |
| Anti-MAL threshold | β < −0.1 | Language classified as Anti-MAL |
| Grey zone | \|β\| ≤ 0.1 | Weak or no MAL effect |
| High compliance ratio | > 0.67 | High MAL compliance |
| Medium compliance ratio | 0.33–0.67 | Medium compliance |
| Low compliance ratio | < 0.33 | Low compliance |
| VO language | VO score > 0.67 | Head-initial word order |
| OV language | VO score < 0.33 | Head-final word order |
| NDO | 0.33 ≤ VO score ≤ 0.67 | No dominant order |

## Setup

### Prerequisites

- Python 3.10+
- [Universal Dependencies treebanks v2.17](https://universaldependencies.org/) (download separately)

### Installation

```bash
git clone https://github.com/YOUR_USERNAME/typometrics.git
cd typometrics
pip install -r requirements.txt
```

### Google Sheets Integration (Optional)

The pipeline uses Google Sheets for language metadata management. To enable this:

1. Create a Google Cloud service account with Sheets API access
2. Copy `credentials.json.template` to `credentials.json`
3. Fill in your service account credentials
4. Share your spreadsheet with the service account email

Without Google Sheets, you can use the local `languageCodes.tsv` and cached `data/metadata.pkl` files.

## Core Modules

### Data I/O

- **`conll.py`** — `Tree` class for dependency tree representation; `conllFile2trees()` for CoNLL-U parsing; `addspan()` for constituent span computation
- **`corpus_prep.py`** — Splits large treebanks into 10k-sentence chunks for parallel processing
- **`data_utils.py`** — Language metadata management via Google Sheets or local files

### Extraction Engine

- **`conll_processing.py`** — The main workhorse. `get_all_stats_parallel()` runs parallel extraction of dependency sizes (exact and partial "anyother" configurations), VO/head-initiality scores, bastard dependencies, and sentence disorder. Uses `multiprocessing` for CPU-bound tasks.

### Analysis

- **`analysis.py`** — Statistical post-processing: min-count filtering, MAL computation, average size derivation
- **`compute_factors.py`** — HCS factors (adjacent position ratios), diagonal growth factors
- **`compute_disorder.py`** — Detection and quantification of non-monotonic constituent ordering

### Verb-Centered Tables

A modular system for generating constituent size tables organized around the verb:

- **`verb_centered_model.py`** — Data models (`CellData`, `FactorData`, `TableConfig`, `TableStructure`)
- **`verb_centered_builder.py`** — Orchestrates table construction
- **`verb_centered_computations.py`** — Marginal means, growth factors, geometric means
- **`verb_centered_formatters.py`** — Output as text, TSV, or Excel
- **`verb_centered_layout.py`** — Column position and symmetry calculations
- **`verb_centered_analysis.py`** — High-level API: `create_verb_centered_table()`, `generate_mass_tables()`

### Visualization

- **`plotting.py`** — MAL curves, heatmaps, 2D scatter plots with regression, HCS plots, head-initiality correlations
- **`generate_html_examples.py`** — Interactive HTML visualizations of configuration examples
- **`generate_ud_maps.py`** — Interactive HTML maps of UD language coverage
- **`mal_html_report.py`** — Interactive HTML reports with Chart.js visualizations; `compute_loglog_regression()` and `compute_decrease_ratio()` are also used by `generate_latex_mal_miniplots.py`

## Key Concepts

- **Dependency positions**: Left dependents are numbered `left_1` (closest to verb), `left_2`, etc. Right dependents similarly `right_1`, `right_2`, etc.
- **Exact configurations**: E.g., `left_2` in a `tot3` (3-dependent) context — exact positional match
- **AnyOther configurations**: Partial configs that aggregate over the opposite side (e.g., `right_1` regardless of left-side count)
- **Bilateral keys**: `bilateral_LxRy` — configurations with exactly x left and y right dependents
- **MAL**: Mean Aggregate Length — average span size across positions for a given valency
- **HCS**: Hierarchical Complexity Score — ratio of position n+1 size to position n size
